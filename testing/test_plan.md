 План тестирования системы приоритизации требований

 1. Цели тестирования
- Проверить корректность работы функционала расчёта приоритета.
- Проверить работу интерфейса и корректное отображение результатов.
- Проверить удобство использования интерфейса тестовыми пользователями.
- Обнаружить ошибки и недочёты в UX.

 2. Объект тестирования
Мини-приложение Telegram для оценки требований: ввод параметров → расчёт приоритета → вывод результата.

 3. Виды тестирования
- Функциональное тестирование
- Пользовательское тестирование (UX)
- Регрессия (минимальная)

 4. Среда тестирования
- Telegram (Desktop + Mobile)
- Python 3.10 / Node.js (если требуется)
- Браузер Google Chrome (для админки)
- OS: Windows 11, Android 11

 5. Инструменты
- Google Sheets (таблицы тест-кейсов)
- Google Forms (сбор обратной связи)
- Telegram Test User
- GitHub issues

 6. Перечень функционала
1. Ввод параметров требования (важность, влияние, сложность).
2. Расчёт итогового приоритета.
3. Отображение результата.
4. Корректность формул.
5. Переключение между разделами.

 7. Критерии приемки
- Все функции работают без критических ошибок.
- Ошибки интерфейса не мешают использованию.
- Результат расчёта корректный.
- Пользователи оценивают удобство не ниже 3.5/5.

## 8. Риски
- Некорректный ввод данных пользователями.
- Ошибки сети Telegram.
